# Architecture

## 定位
`architecture` 用来沉淀高并发、微服务、分布式系统、网关、中间件和系统设计层面的内容。

## 收录范围
适合放服务拆分、分布式一致性、网关设计、链路追踪、限流熔断、消息系统、系统容量规划等文章。
不适合放具体语言实现细节和单点数据库调优，这些更适合放到 `backend` 或 `database`。

## SEO 方向
主关键词：系统设计、微服务架构、高并发架构、分布式系统、API 网关、服务治理。
长尾关键词：etcd 原理、Kong 教程、OpenTracing 使用、高并发缓存设计、分布式拆分问题、Kafka 架构。
常见搜索意图：架构选型、原理学习、系统扩展、性能瓶颈处理、面试和复盘。

## 专题建议
适合持续沉淀为微服务拆分、高并发架构演进、网关治理、Kafka 与消息队列、可观测性专题。

## 写作提示
标题尽量体现场景和问题，例如“服务化拆分带来的问题与解决思路”。
正文建议多用架构图、分层视角、利弊对比和演进路径，增强专题沉淀能力。
