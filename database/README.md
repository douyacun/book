# Database

## 定位
`database` 用来沉淀数据库与搜索系统相关内容，重点覆盖 MySQL、Redis、Elasticsearch、数据结构、查询优化和存储原理。

## 收录范围
适合放索引、事务、锁、执行计划、缓存结构、分布式存储、ES 检索机制和数据库调优类文章。
不适合放纯业务代码、部署脚本和服务器运维内容，这些应分别进入 `backend` 或 `devops`。

## SEO 方向
主关键词：MySQL 优化、Redis 原理、Elasticsearch 教程、数据库索引、SQL 调优、事务隔离。
长尾关键词：MySQL explain、索引失效、InnoDB 原理、Redis skiplist、Redis SDS、ES mapping、ES 分词。
常见搜索意图：性能优化、原理学习、面试准备、故障排查、底层数据结构分析。

## 专题建议
可以围绕 MySQL 性能优化、Redis 数据结构、Elasticsearch 检索原理、数据库面试专题做持续输出。

## 写作提示
标题最好直接落在搜索词上，例如“MySQL 索引失效的常见场景”。
正文建议加入执行计划、结构图、示例 SQL 和对比结论，并补充站内相关文章内链。
