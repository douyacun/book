# Backend

## 定位
`backend` 用来沉淀服务端开发相关内容，重点覆盖 Go、PHP、接口设计、性能优化、工程实践和线上问题处理。

## 收录范围
适合放语言特性、并发模型、网络编程、框架使用、调试排障、服务治理、性能分析等文章。
不适合放数据库原理、运维部署、纯前端实现和泛运营内容，这些应分别进入 `database`、`devops`、`frontend`、`growth`。

## SEO 方向
主关键词：Go 教程、Go 并发、Go 内存分配、Go 调试、PHP 实战、接口设计、服务端性能优化。
长尾关键词：Go channel 原理、Go GC 分析、grpc 入门、protobuf 使用、golang pprof、远程调试、连接池优化。
常见搜索意图：原理解释、问题排查、面试整理、线上故障解决、最佳实践总结。

## 专题建议
可以持续沉淀为 Go 并发、Go 内存管理、RPC 与微服务通信、线上排障、服务端工程化几个系列。

## 写作提示
标题尽量体现技术对象和结果，例如“Go channel 的底层实现与使用误区”。
正文优先用“问题背景、原理、示例、排查、结论”的结构，并补站内链接到相关协议、架构和数据库文章。
